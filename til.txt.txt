1.앞으로 git 프로젝트로 만들고 싶은 것

항해 99를 시작하면서 하는 팀 프로젝트의 협업 및 개인 프로젝트들을 깃을 통해 관리를 하고 깃헙(원격repo)에 업데이트를 하여 효율적으로
소스코드들을 관리해보려 한다. 또한 오픈되어 있는 다른 개발자들의 프로젝트 파일을 클론해와서 코드들을 보며 공부를 해볼 수 있을 것 같다!

2. 1주차 주요 개념 키워드 적어보기

1)SourceTree란?
Git을 쉽게 사용할 수 있게 해주는 도구이다.

2)버전관리란?
버전관리란 진행하고 있는 프로젝트의 상태(누가, 언제, 무엇을)에 대한 정보를 남기는 것!

3)commit 이란?
commit은 git에 내가 현재 진행하고 있는 프로젝트의 상태를 저장하는것이다.(마치 사진을 찍어 저장을 하는 것과 같음!)
commit을 하여 나의 local repo에 저장을 하고 수정사항을 관리할 수 있음!
빨간색 - 는 삭제된 내용, 초록색 +는 추가된 내용.

4)commit history란?
commit log라도 불리우며 내가 commit 한 파일들을 commit한 순서대로 볼 수 있고 수정된 사항들도 확인이 가능하다.
(프로젝트 진행시 진전된 사항, 추가 또는 수정된 사항들을 그때 그때 commit 하고 만약 버그 시 오류가 나기 이전의 코드들과 비교를 해볼 수 있고
기능 추가 및 수정 이전 상황으로 되돌릴 때 사용하기 좋을 것 같다.)

5)로컬 repo와 원격 repo(local repository & remote repository)
로컬 repo란 내 컴퓨터에 저장되어 져 있는 git으로 관리되어지는 프로젝트를 뜻한다.
원격 repo란 github처럼 어디서든 접속이 가능한 곳에 저장되어져 있는 것을 말한다.

6)tracking 이란?
트래킹이란 로컬 repo와 원격 repo 를 연결하는 것을 뜻한다.

7)Push & Pull
push는 로컬 repo를 원격 repo로 보내는 것을 뜻하고 pull을 원격 repo를 로컬 repo로 가져오는 것을 말한다.  

8)로컬 repo와 원격 repo 연동하기
만약 github에  원격 repo 가 없다면 먼저 원격 repo를 생성한 후 로컬 repo와 tracking 을 통해 연결한다.
일반적으로 충돌(원격과 로컬이 같은 파일을 수정 할 경우 발생: 이러한 경우 git에서 최종적으로 어떤 파일을 최종으로 할 지 확인 메세지를 줌)이 발생하는 것을 막기 위해 원격 repo를 먼저 로컬 repo로 pull 해오고 commit을 진행한 후 다시 원격 repo로 push를 하는 방법을 사용하는 것이 좋다!
### Pull > Commit > Push 순으로 진행! 

9)클론(Clone)이란?
말 그대로 복제를 하는 것을 뜻하고 원격 repo를(내가 다른 환경에서 작업한 것을 원격 repo에 올린 것 또는 다른 사람의 원격 repo)를 다운받아 내컴퓨터에서 보고자 할 때 하는 것을 말한다.
